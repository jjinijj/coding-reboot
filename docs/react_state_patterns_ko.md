# React 상태 설계 패턴

목표: **최소한의 state 유지**

------------------------------------------------------------------------

## 1. Single Source of Truth

데이터는 하나의 state만 사용

------------------------------------------------------------------------

## 2. Derived State

계산 가능한 값은 state로 저장하지 않는다

    const filteredItems = items.filter(...)

------------------------------------------------------------------------

## 3. Local UI State

UI 상태는 컴포넌트 내부에 유지

예

-   modal open
-   tab 선택

------------------------------------------------------------------------

## 4. Lift State Up

여러 컴포넌트가 사용하면 부모가 관리

------------------------------------------------------------------------

## 5. Colocate State

state는 **사용 위치 가까이**

------------------------------------------------------------------------

## 6. Reducer Pattern

조건

-   상태 변경이 많음
-   로직 복잡

------------------------------------------------------------------------
